# 3.2 - Ejecutar pruebas de carga basica y estabilidad

## Objetivo

Realizar pruebas de carga ligera y estabilidad para endpoints y flujos de eventos/notificaciones mas criticos.

## Razon Tecnica

Antes de cierre post-MVP se necesita una linea base de comportamiento bajo carga moderada.

## Comandos Exactos

```bash
./mvnw test -Dspring.profiles.active=test
curl -s http://localhost:8080/actuator/metrics | rg "http.server.requests|jvm.memory.used"
```

## Ejemplo De Codigo (Si Aplica)

```txt
Escenarios minimos:
- pico corto de creacion de tickets
- rafaga de eventos de prioridad
- rafaga de notificaciones in-app/realtime
```

## Verificacion

1. El sistema mantiene tiempos aceptables en carga base definida.
2. No hay crecimiento anomalo de memoria o errores 5xx sostenidos.
3. Se registran metricas y hallazgos para tuning posterior.

## Resultado Esperado

Linea base de rendimiento y estabilidad documentada para operacion.

## Criterio De Cierre

- [ ] objetivo cumplido
- [ ] verificacion ejecutada
- [ ] evidencia registrada
- [ ] documentacion relacionada actualizada

## Troubleshooting Breve (Si Aplica)

- latencia alta en endpoints criticos -> revisar consultas y serializacion de payload
- consumo de memoria creciente -> inspeccionar colas en memoria y listeners no drenados
