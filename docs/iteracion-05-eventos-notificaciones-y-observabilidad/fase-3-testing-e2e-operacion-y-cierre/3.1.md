# 3.1 - Ejecutar suite E2E cross-modulo

## Objetivo

Ejecutar suite E2E de flujos cross-modulo que involucren auth, account, ticket, product y notification.

## Razon Tecnica

Los riesgos principales aparecen en la integracion entre modulos, no solo en pruebas unitarias aisladas.

## Comandos Exactos

```bash
./mvnw test -Dspring.profiles.active=test -Dtest='*E2E*Test,*Integration*Test'
./mvnw -DskipTests compile
```

## Ejemplo De Codigo (Si Aplica)

```txt
Flujos minimos:
- mensaje externo -> conversion a ticket -> notificacion
- cambio de prioridad ticket -> evento -> notificacion realtime
- accion de soporte -> auditoria -> consulta de historial
```

## Verificacion

1. Flujos E2E criticos terminan en estado esperado.
2. Se confirma emision/consumo de eventos relevantes.
3. No hay regresiones en contratos API existentes.

## Resultado Esperado

Cobertura E2E basica para validar el comportamiento real del sistema modular.
