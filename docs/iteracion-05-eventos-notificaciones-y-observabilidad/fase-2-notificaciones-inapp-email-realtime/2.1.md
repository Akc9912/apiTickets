# 2.1 - Implementar modulo notification base

## Objetivo

Crear modulo notification con modelo de dominio, persistencia y casos de uso base desacoplados de modulos emisores.

## Razon Tecnica

Las notificaciones requieren su propio contexto para evitar que ticket/product concentren logica transversal.

## Comandos Exactos

```bash
mkdir -p src/main/java/com/poo/miapi/module/notification/{domain,application,presentation,persistence,infrastructure}
./mvnw -DskipTests compile
rg "notification|template|channel" src/main/java -g "*.java"
```
