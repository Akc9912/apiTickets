# 2.3 - Implementar canal realtime y preferencias usuario

## Objetivo

Agregar canal realtime por websocket y aplicar preferencias de usuario para controlar volumen y tipo de notificaciones.

## Razon Tecnica

Sin preferencias se genera ruido; sin realtime se pierde capacidad de respuesta en casos criticos.

## Comandos Exactos

```bash
./mvnw -DskipTests compile
./mvnw test -Dspring.profiles.active=test -Dtest='*WebSocket*Test,*NotificationPreference*Test'
rg "websocket|stomp|preference|notification" src/main/java -g "*.java"
```

## Ejemplo De Codigo (Si Aplica)

```txt
Reglas sugeridas:
- usuario puede desactivar EMAIL para eventos LOW
- eventos CRITICAL siempre notifican in-app + realtime
- realtime publica en canal por usuario/rol
```

## Verificacion

1. Eventos elegibles se emiten en canal realtime correcto.
2. Preferencias de usuario se respetan por canal y severidad.
3. Configuracion por defecto evita perdida de eventos criticos.

## Resultado Esperado

Notificaciones en tiempo real controladas por preferencias y listas para operacion diaria.
