# 2.2 - Implementar canales in-app y email

## Objetivo

Implementar envio de notificaciones por canales in-app y email con plantillas versionadas.

## Razon Tecnica

Son los canales de mayor valor inmediato para operacion y comunicacion con usuario.

## Comandos Exactos

```bash
./mvnw -DskipTests compile
./mvnw test -Dspring.profiles.active=test -Dtest='*Notification*Test,*Email*Test'
rg "mail|smtp|notification" src/main/resources src/main/java -g "*"
```

## Ejemplo De Codigo (Si Aplica)

```txt
Canales:
- IN_APP -> persistencia + consulta de bandeja
- EMAIL -> envio async con plantilla

Eventos disparadores:
- TicketCreatedEvent
- TicketPriorityChangedEvent
```

## Verificacion

1. Notificacion in-app queda disponible para consulta por usuario.
2. Email se genera con plantilla y datos de contexto.
3. Fallo de un canal no rompe procesamiento de otros canales.

## Resultado Esperado

Canales in-app y email operativos con flujo basico multi-canal.
