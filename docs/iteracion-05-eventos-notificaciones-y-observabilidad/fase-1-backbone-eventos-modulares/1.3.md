# 1.3 - Aplicar idempotencia y manejo de errores en eventos

## Objetivo

Agregar controles de idempotencia, reintento y manejo de errores para procesamiento de eventos internos.

## Razon Tecnica

Procesamiento de eventos sin protecciones genera duplicidad funcional y flujos no deterministas.

## Comandos Exactos

```bash
./mvnw -DskipTests compile
./mvnw test -Dspring.profiles.active=test -Dtest='*Idempotency*Test,*Event*Retry*Test'
rg "idempot|retry|dead letter|error" src/main/java -g "*.java"
```

## Ejemplo De Codigo (Si Aplica)

```txt
Reglas minimas:
- eventId unico por publicacion
- tabla/registro de eventos procesados
- politica de reintento acotada
- log estructurado en error terminal
```

## Verificacion

1. El mismo evento no se procesa dos veces con efecto funcional.
2. Errores transitorios tienen reintento controlado.
3. Errores terminales quedan registrados para operacion.

## Resultado Esperado

Procesamiento de eventos estable, trazable y resistente a fallos transitorios.
