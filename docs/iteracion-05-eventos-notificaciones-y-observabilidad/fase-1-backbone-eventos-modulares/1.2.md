# 1.2 - Implementar publisher/listeners base

## Objetivo

Implementar infraestructura base de publicacion y consumo de eventos dentro del monolito modular.

## Razon Tecnica

El catalogo sin mecanismo de publicacion/consumo no habilita integracion real entre modulos.

## Comandos Exactos

```bash
./mvnw -DskipTests compile
./mvnw test -Dspring.profiles.active=test -Dtest='*Event*Test,*Listener*Test'
rg "ApplicationEventPublisher|@EventListener|publish" src/main/java -g "*.java"
```

## Ejemplo De Codigo (Si Aplica)

```txt
Flujo base:
- modulo A publica DomainEvent
- modulo B escucha IntegrationEvent
- handler aplica accion idempotente
```

## Verificacion

1. Publicadores emiten eventos en puntos de dominio definidos.
2. Listeners procesan eventos sin acoplar capas de presentation.
3. Fallas de listener no rompen transaccion principal cuando no corresponde.

## Resultado Esperado

Backbone de eventos funcional para orquestar capacidades cross-modulo.
