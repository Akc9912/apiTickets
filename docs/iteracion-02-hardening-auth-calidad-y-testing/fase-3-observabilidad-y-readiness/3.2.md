# 3.2 - Estandarizar logging estructurado y correlacion

## Objetivo

Estandarizar formato de logs y agregar correlacion de requests/eventos para diagnostico rapido.

## Razon Tecnica

Logs sin estructura dificultan troubleshooting y analisis de incidentes.

## Comandos Exactos

```bash
rg "logging\.pattern|logging\.level" src/main/resources/application.properties -g "*"
./mvnw -DskipTests compile
```

## Ejemplo De Codigo (Si Aplica)

```txt
campos minimos en log:
- timestamp
- level
- service
- traceId/correlationId
- endpoint
- outcome
```

## Verificacion

1. Logs siguen formato consistente en todos los modulos MVP.
2. Cada request tiene identificador de correlacion.
3. Se puede trazar flujo entre auth/account/ticket.

## Resultado Esperado

Observabilidad de logs usable para soporte tecnico y analisis operativo.

## Criterio De Cierre

- [ ] objetivo cumplido
- [ ] verificacion ejecutada
- [ ] evidencia registrada
- [ ] documentacion relacionada actualizada

## Troubleshooting Breve (Si Aplica)

- logs demasiado verbosos -> ajustar niveles por paquete
- correlacion perdida en async -> propagar contexto en executors
