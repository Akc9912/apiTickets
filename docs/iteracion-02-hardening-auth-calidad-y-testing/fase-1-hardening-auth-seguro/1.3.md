# 1.3 - Registrar auditoria de eventos de seguridad

## Objetivo

Registrar eventos de seguridad criticos para trazabilidad operativa y soporte forense basico.

## Razon Tecnica

Hardening sin auditoria deja baja capacidad de diagnostico ante incidentes.

## Comandos Exactos

```bash
mkdir -p logs/security
./mvnw -DskipTests compile
```

## Ejemplo De Codigo (Si Aplica)

```txt
eventos minimos:
- login_success
- login_failed
- recovery_requested
- recovery_confirmed
- rate_limit_triggered
```

## Verificacion

1. Se registran eventos con timestamp, actor y origen.
2. No se loguean secretos ni tokens completos.
3. Se puede filtrar por tipo de evento y usuario.

## Resultado Esperado

Auditoria de seguridad util para troubleshooting y seguimiento operativo.

## Criterio De Cierre

- [ ] objetivo cumplido
- [ ] verificacion ejecutada
- [ ] evidencia registrada
- [ ] documentacion relacionada actualizada

## Troubleshooting Breve (Si Aplica)

- exceso de ruido en logs -> aplicar niveles y campos minimos obligatorios
- datos sensibles en log -> sanitizar payload antes de persistir
