# 1.1 - Definir flujo seguro de verificacion y recuperacion

## Objetivo

Definir e implementar flujo seguro para verificacion de cuenta y recuperacion de acceso usando Supabase como autoridad de identidad.

## Razon Tecnica

Post-MVP se necesita reducir soporte manual y riesgos de cuentas vulnerables sin volver a gestionar credenciales locales en backend.

## Comandos Exactos

```bash
rg "auth|reset|verify|supabase" src/main/java -g "*.java"
./mvnw -DskipTests compile
```

## Ejemplo De Codigo (Si Aplica)

```txt
POST /api/v1/auth/request-account-verification
POST /api/v1/auth/request-password-recovery
POST /api/v1/auth/confirm-recovery
```

## Verificacion

1. Los endpoints de verificacion/recuperacion existen y validan input.
2. El backend delega identidad en Supabase.
3. No se reintroduce emision local de credenciales.

## Resultado Esperado

Flujo de seguridad de cuenta definido y operativo sin romper arquitectura modular.

## Criterio De Cierre

- [ ] objetivo cumplido
- [ ] verificacion ejecutada
- [ ] evidencia registrada

## Troubleshooting Breve (Si Aplica)

- error de integracion con Supabase -> validar keys, issuer y permisos de API
- flujo ambiguo entre account/auth -> mantener responsabilidad de identidad solo en auth
