# 1.2 - Implementar rate limiting y politicas anti-abuso

## Objetivo

Agregar protecciones anti-abuso para endpoints sensibles de autenticacion y recuperacion.

## Razon Tecnica

Sin controles de frecuencia, el sistema queda expuesto a fuerza bruta y spam de recuperacion.

## Comandos Exactos

```bash
./mvnw -DskipTests compile
rg "auth" src/main/java/com/poo/miapi/module/auth -g "*.java"
```

## Ejemplo De Codigo (Si Aplica)

```txt
limite sugerido inicial:
- login: 5 intentos / 10 min por usuario+ip
- recover: 3 solicitudes / 30 min por usuario
```

## Verificacion

1. Endpoints sensibles aplican limite por ventana temporal.
2. Exceso de intentos responde con codigo y mensaje consistente.
3. Politicas quedan documentadas en config.

## Resultado Esperado

Reduccion de superficie de abuso con politicas simples y medibles.
