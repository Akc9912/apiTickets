# 2.2 - Implementar suite de contrato y regresion MVP

## Objetivo

Construir una suite inicial de pruebas de contrato API y regresion para auth, account y ticket.

## Razon Tecnica

Despues del MVP se necesita congelar comportamiento esperado antes de agregar nuevas features.

## Comandos Exactos

```bash
./mvnw test -Dspring.profiles.active=test
curl -s http://localhost:8080/v3/api-docs > /tmp/openapi-current.json
```

## Ejemplo De Codigo (Si Aplica)

```txt
casos minimos:
- auth: token valido/invalido
- account: lectura de perfil por rol
- ticket: crear/listar/reabrir con permisos
```

## Verificacion

1. La suite de contrato API cubre endpoints criticos MVP.
2. La regression suite detecta cambios de comportamiento funcional.
3. Los resultados quedan versionados como evidencia.

## Resultado Esperado

Red de seguridad para evolucionar funcionalidades sin romper el core MVP.

## Criterio De Cierre

- [ ] objetivo cumplido
- [ ] verificacion ejecutada
- [ ] evidencia registrada
- [ ] documentacion relacionada actualizada

## Troubleshooting Breve (Si Aplica)

- tests fragiles por datos dinamicos -> usar fixtures controlados y clock estable
- fallas no deterministas -> aislar dependencias externas con mocks/contratos
