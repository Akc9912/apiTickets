# 2.3 - Activar quality gates en pipeline CI

## Objetivo

Configurar gates de calidad minima en CI para evitar merge de cambios sin build/test validos.

## Razon Tecnica

Sin gates automatizados el estandar de calidad depende de revision manual y se degrada rapido.

## Comandos Exactos

```bash
./mvnw -DskipTests compile
./mvnw test -Dspring.profiles.active=test
```

## Ejemplo De Codigo (Si Aplica)

```txt
gates minimos:
- compile obligatorio
- test suite obligatoria
- contrato API smoke
```

## Verificacion

1. El pipeline falla cuando no pasa compile o tests.
2. El pipeline publica resultados de pruebas.
3. El merge requiere gates en verde.

## Resultado Esperado

Disciplina de calidad sostenida por automatizacion.

## Criterio De Cierre

- [ ] objetivo cumplido
- [ ] verificacion ejecutada
- [ ] evidencia registrada
- [ ] documentacion relacionada actualizada

## Troubleshooting Breve (Si Aplica)

- pipeline demasiado lento -> separar jobs por tipo de prueba
- flakiness en CI -> aislar tests inestables y corregir origen
