# 7.2 - Limpiar residuos legacy y rutas duplicadas

## Objetivo

Eliminar residuos tecnicos legacy y rutas duplicadas que ya no corresponden al flujo modular final.

## Razon Tecnica

Conservar codigo legacy activo despues del cutover aumenta deuda y genera ambiguedad operativa. Esta tarea consolida una sola ruta de verdad tecnica.

## Comandos Exactos

```bash
rg "legacy|deprecated|module.user" src/main/java -g "*.java"
rg "@RequestMapping\(" src/main/java/com/poo/miapi/module -g "*Controller.java"
./mvnw -DskipTests compile
./mvnw test -Dspring.profiles.active=test
```

## Ejemplo De Codigo (Si Aplica)

```txt
// remover controller duplicado legado cuando exista endpoint equivalente nuevo
```

## Verificacion

1. No hay rutas duplicadas para misma capacidad de negocio.
2. Se reducen referencias activas a paquetes legacy en flujo MVP.
3. Build y tests continúan estables despues de limpieza.

## Resultado Esperado

Codigo operativo simplificado, sin ambiguedad entre flujo nuevo y legado.
