# 7.1 - Ejecutar cutover al flujo modular nuevo

## Objetivo

Activar operativamente el flujo modular nuevo para auth, account y ticket como ruta principal de backend.

## Razon Tecnica

El valor de la migracion aparece cuando el sistema usa efectivamente el flujo nuevo. Esta tarea materializa el cambio de arquitectura en operacion.

## Comandos Exactos

```bash
./mvnw -DskipTests compile
./mvnw spring-boot:run
curl -s http://localhost:8080/actuator/health
rg "@RequestMapping\(" src/main/java/com/poo/miapi/module -g "*Controller.java"
```

## Ejemplo De Codigo (Si Aplica)

```txt
/health => {"status":"UP"}
```

## Verificacion

1. La aplicacion arranca con el flujo modular nuevo activo.
2. Endpoints principales de auth/account/ticket responden correctamente.
3. No hay dependencia funcional obligatoria en rutas legacy para flujo MVP.

## Resultado Esperado

Cutover ejecutado de forma controlada con backend funcionando sobre arquitectura objetivo.
