# 6.1 - Verificar limites modulares y dependencias

## Objetivo

Comprobar que el codigo migrado respeta limites de modulo y no introduce dependencias directas prohibidas.

## Razon Tecnica

El manifiesto exige modularidad y bajo acoplamiento. Esta tarea valida cumplimiento tecnico antes del cutover.

## Comandos Exactos

```bash
rg "import com\.poo\.miapi\.module\.user\." src/main/java/com/poo/miapi/module -g "*.java"
rg "import com\.poo\.miapi\.module\.(auth|account|ticket)\..*\.repository\." src/main/java/com/poo/miapi/module -g "*.java"
./mvnw -DskipTests compile
```

## Ejemplo De Codigo (Si Aplica)

```txt
// permitido: consumo por contrato
import com.poo.miapi.shared.contracts.account.AccountQueryContract;
```

## Verificacion

1. No hay imports prohibidos en modulos migrados.
2. Integraciones intermodulo pasan por shared/contracts.
3. Build se mantiene estable luego de limpieza de dependencias.

## Resultado Esperado

Limites modulares verificados con evidencia automatizable por busqueda de imports.

## Criterio De Cierre

- [ ] objetivo cumplido
- [ ] verificacion ejecutada
- [ ] evidencia registrada
- [ ] documentacion relacionada actualizada

## Troubleshooting Breve (Si Aplica)

- falso positivo en archivos legacy -> restringir chequeo a paquetes nuevos de iteracion
- contrato faltante -> crear interfaz en shared/contracts antes de continuar
