# 0.1 - Levantar linea base tecnica

## Objetivo

Levantar una foto tecnica inicial del estado real del backend antes de cambios estructurales, incluyendo compilacion, tests y arranque de aplicacion.

## Razon Tecnica

Sin baseline verificable no se puede medir regression ni validar que la migracion mejora el sistema. Esta tarea reduce riesgo de cambios ciegos.

## Comandos Exactos

```bash
./mvnw -DskipTests compile
./mvnw test -Dspring.profiles.active=test
./mvnw spring-boot:run
```

## Ejemplo De Codigo (Si Aplica)

```txt
No aplica. Esta tarea es de levantamiento tecnico y evidencia operativa.
```

## Verificacion

1. El comando de compilacion termina con BUILD SUCCESS.
2. El comando de tests ejecuta suite sin errores bloqueantes de contexto.
3. Con la app levantada, /actuator/health y /v3/api-docs responden.

## Resultado Esperado

Existe una linea base clara del estado tecnico inicial para comparar cada fase de la iteracion.

## Criterio De Cierre

- [ ] objetivo cumplido
- [ ] verificacion ejecutada
- [ ] evidencia registrada
- [ ] documentacion relacionada actualizada

## Troubleshooting Breve (Si Aplica)

- fallo por variables de entorno -> validar archivo .env y propiedades DB_URL, DB_USER, DB_PASS
- fallo en tests por perfil -> ejecutar con -Dspring.profiles.active=test
