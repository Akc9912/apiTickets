# 0.2 - Corregir bloqueantes de compilacion

## Objetivo

Eliminar errores de compilacion y stubs bloqueantes para estabilizar el punto de partida de la migracion.

## Razon Tecnica

Migrar arquitectura sobre codigo que no compila amplifica el riesgo y dificulta aislar fallas. Esta tarea asegura una base tecnicamente utilizable.

## Comandos Exactos

```bash
./mvnw -DskipTests compile
rg "UnsupportedOperationException|TODO Auto-generated method stub" src/main/java
./mvnw -DskipTests compile
```

## Ejemplo De Codigo (Si Aplica)

```txt
// anti-patron a remover
throw new UnsupportedOperationException("Unimplemented method");
```

## Verificacion

1. No quedan errores de compilacion en los modulos de la iteracion.
2. Los stubs bloqueantes fueron implementados o aislados con criterio tecnico.
3. El segundo compile confirma baseline estable.

## Resultado Esperado

El backend compila de forma consistente y deja de depender de metodos sin implementar en flujos criticos.
