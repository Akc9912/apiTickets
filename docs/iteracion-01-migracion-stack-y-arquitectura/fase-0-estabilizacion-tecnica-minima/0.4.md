# 0.4 - Cerrar baseline con evidencia tecnica

## Objetivo

Cerrar formalmente la fase de baseline con evidencia reproducible de compilacion, tests y estado del repositorio.

## Razon Tecnica

La iteracion requiere trazabilidad. Esta tarea documenta el punto de partida y evita discusiones sobre estado inicial no registrado.

## Comandos Exactos

```bash
mkdir -p docs/iteracion-01-migracion-stack-y-arquitectura/fase-0-estabilizacion-tecnica-minima/evidencia
./mvnw -DskipTests compile > docs/iteracion-01-migracion-stack-y-arquitectura/fase-0-estabilizacion-tecnica-minima/evidencia/compile-baseline.log
./mvnw test -Dspring.profiles.active=test > docs/iteracion-01-migracion-stack-y-arquitectura/fase-0-estabilizacion-tecnica-minima/evidencia/test-baseline.log
git status --short > docs/iteracion-01-migracion-stack-y-arquitectura/fase-0-estabilizacion-tecnica-minima/evidencia/git-status-baseline.txt
```

## Ejemplo De Codigo (Si Aplica)

```txt
[INFO] BUILD SUCCESS
```

## Verificacion

1. Existen los 3 archivos de evidencia (compile, test, git status).
2. Los logs muestran ejecucion completa de comandos.
3. El README de fase puede marcarse como verificable por evidencia.

## Resultado Esperado

La fase 0 queda cerrada con material auditable para comparar cambios en toda la iteracion.

## Criterio De Cierre

- [ ] objetivo cumplido
- [ ] verificacion ejecutada
- [ ] evidencia registrada
- [ ] documentacion relacionada actualizada

## Troubleshooting Breve (Si Aplica)

- logs vacios o truncados -> relanzar comandos sin redireccion y validar permisos de escritura
- git status incluye cambios no esperados -> documentar si son previos a la iteracion
