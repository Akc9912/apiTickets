# 5.3 - Adaptar endpoints y DTOs de ticket

## Objetivo

Actualizar endpoints y DTOs de ticket al modelo desacoplado y nomenclatura consistente.

## Razon Tecnica

El contrato API actual tiene inconsistencias de naming y acoplamiento de DTOs a user legacy. Esta tarea normaliza contrato y reduce friccion en frontend.

## Comandos Exactos

```bash
rg "tittle|statis|Developer;|UserResponseDto" src/main/java/com/poo/miapi/module/ticket -g "*.java"
./mvnw -DskipTests compile
./mvnw test -Dspring.profiles.active=test
```

## Ejemplo De Codigo (Si Aplica)

```txt
public class TicketResponseDto {
    private UUID id;
    private String title;
    private String status;
    private UUID creatorAccountId;
}
```

## Verificacion

1. DTOs de ticket usan naming coherente y sin typo tecnico.
2. Endpoints reflejan payload desacoplado de entidades user.
3. Tests de contrato API pasan para rutas de ticket.

## Resultado Esperado

Contrato ticket limpio, consistente y alineado al backend modular.
