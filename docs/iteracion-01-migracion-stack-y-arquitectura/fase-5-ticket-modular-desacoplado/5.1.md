# 5.1 - Refactor ticket a arquitectura por capas

## Objetivo

Refactorizar el modulo ticket a la estructura por capas definida por engineering.

## Razon Tecnica

Ticket hoy mezcla responsabilidades y depende de modelos externos. Esta tarea ordena el modulo para habilitar desacople real.

## Comandos Exactos

```bash
mkdir -p src/main/java/com/poo/miapi/module/ticket/{presentation/{controller,dto},application/{usecase,service,dto},domain/{model,valueobject,event,policy},persistence/{repository,query,entity},infrastructure/{config,mapper,client}}
find src/main/java/com/poo/miapi/module/ticket -maxdepth 3 -type d | sort
./mvnw -DskipTests compile
```

## Ejemplo De Codigo (Si Aplica)

```txt
module/ticket/application/service/TicketApplicationService.java
```

## Verificacion

1. Ticket presenta capas separadas con responsabilidades claras.
2. Controllers usan servicios de application, no repositories.
3. Compilacion sin errores tras reorganizacion.

## Resultado Esperado

Modulo ticket preparado para aplicar desacople de user/account y migracion de contratos.
