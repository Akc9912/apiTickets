# 5.2 - Eliminar acoplamientos directos ticket-user

## Objetivo

Eliminar dependencias directas de ticket hacia entidades y servicios de user legacy.

## Razon Tecnica

La regla tecnica del proyecto exige que codigo nuevo no interactue con codigo legacy. Esta tarea cumple ese limite y evita dependencia circular.

## Comandos Exactos

```bash
rg "import com\.poo\.miapi\.module\.user\." src/main/java/com/poo/miapi/module/ticket -g "*.java"
rg "@ManyToOne|@JoinColumn" src/main/java/com/poo/miapi/module/ticket -g "*.java"
./mvnw -DskipTests compile
```

## Ejemplo De Codigo (Si Aplica)

```txt
private UUID creatorAccountId;
private UUID assignedDeveloperAccountId;
```

## Verificacion

1. No quedan imports directos a module.user en ticket nuevo.
2. Relacion ticket-account se maneja por IDs y contratos.
3. Build permanece en verde luego del desacople.

## Resultado Esperado

Ticket desacoplado de user legacy y alineado con integracion por contratos.
