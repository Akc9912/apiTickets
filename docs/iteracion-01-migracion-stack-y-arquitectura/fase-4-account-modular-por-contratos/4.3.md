# 4.3 - Publicar contratos account para otros modulos

## Objetivo

Publicar contratos de account para consulta de perfil y validacion de estado/rol desde otros modulos.

## Razon Tecnica

Ticket y otros modulos deben consumir capacidades de account sin importar su implementacion interna.

## Comandos Exactos

```bash
mkdir -p src/main/java/com/poo/miapi/shared/contracts/account
touch src/main/java/com/poo/miapi/shared/contracts/account/AccountQueryContract.java
touch src/main/java/com/poo/miapi/shared/contracts/account/AccountStatusContract.java
rg "module.user" src/main/java/com/poo/miapi/module/ticket -g "*.java"
```

## Ejemplo De Codigo (Si Aplica)

```txt
public interface AccountStatusContract {
    boolean isActive(UUID accountId);
    boolean hasRole(UUID accountId, String role);
}
```

## Verificacion

1. Existen contratos account en shared/contracts/account.
2. Ticket puede referenciar contrato sin importar clases account internas.
3. Se reduce dependencia directa a module.user legacy.

## Resultado Esperado

Integracion intermodulo por interfaces estables y versionables.
