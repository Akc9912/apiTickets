# 4.2 - Migrar perfil de usuario a account

## Objetivo

Migrar datos y casos de uso de perfil al modulo account, manteniendo fuera de account la gestion de credenciales.

## Razon Tecnica

El modelo legacy user mezcla perfil, auth y roles. Esta tarea separa responsabilidades para habilitar evolucion modular.

## Comandos Exactos

```bash
rg "module.user.model.User|UserRepository|/api/user" src/main/java -g "*.java"
./mvnw -DskipTests compile
./mvnw test -Dspring.profiles.active=test
```

## Ejemplo De Codigo (Si Aplica)

```txt
public class AccountProfile {
    private UUID accountId;
    private String email;
    private String displayName;
    private boolean active;
}
```

## Verificacion

1. Account expone perfil sin campos de password ni logica de login.
2. Endpoints de perfil apuntan a servicios de account en codigo nuevo.
3. Build y tests basicos siguen en verde.

## Resultado Esperado

Perfil de usuario desacoplado de autenticacion y preparado para consumo por ticket.
