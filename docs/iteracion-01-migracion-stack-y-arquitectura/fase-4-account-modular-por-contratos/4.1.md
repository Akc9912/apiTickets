# 4.1 - Crear estructura por capas de account

## Objetivo

Crear el modulo account con estructura por capas para manejar perfil de usuario desacoplado de autenticacion.

## Razon Tecnica

Separar identidad de perfil evita mezclar auth con dominio de usuario y cumple el principio de contratos claros entre modulos.

## Comandos Exactos

```bash
mkdir -p src/main/java/com/poo/miapi/module/account/{presentation/{controller,dto},application/{usecase,service,dto},domain/{model,valueobject,event,policy},persistence/{repository,query,entity},infrastructure/{config,mapper,client}}
find src/main/java/com/poo/miapi/module/account -maxdepth 3 -type d | sort
./mvnw -DskipTests compile
```

## Ejemplo De Codigo (Si Aplica)

```txt
module/account/domain/model/AccountProfile.java
```

## Verificacion

1. El modulo account contiene las 5 capas definidas.
2. No se colocan clases de seguridad en account.
3. Compilacion permanece estable tras crear estructura.

## Resultado Esperado

Account listo para recibir migracion de perfil y exponer contratos de consulta.
