# 3.3 - Reemplazar filtro y principal de seguridad

## Objetivo

Actualizar el filtro de seguridad para construir principal autenticado desde claims del token Supabase, desacoplado de user legacy.

## Razon Tecnica

La seguridad actual depende de UserDetails local acoplado a module/user. Esta tarea permite autenticar por contrato de identidad y preparar account modular.

## Comandos Exactos

```bash
rg "JwtAuthenticationFilter|UserDetailsService|AuthenticationPrincipal" src/main/java -g "*.java"
rg "module.user" src/main/java/com/poo/miapi/shared/security -g "*.java"
./mvnw -DskipTests compile
./mvnw test -Dspring.profiles.active=test
```

## Ejemplo De Codigo (Si Aplica)

```txt
Authentication auth = new UsernamePasswordAuthenticationToken(principal, null, authorities);
SecurityContextHolder.getContext().setAuthentication(auth);
```

## Verificacion

1. El filtro nuevo no requiere cargar usuario legacy para autenticar token.
2. El principal se construye desde claims validados.
3. Endpoints protegidos reconocen autenticacion y roles.

## Resultado Esperado

Seguridad desacoplada del modelo legacy, lista para integrarse con account por contratos.
