# 3.2 - Implementar validacion JWT Supabase

## Objetivo

Implementar en auth la validacion de JWT emitido por Supabase, eliminando emision local de token en backend.

## Razon Tecnica

El stack oficial define Supabase Auth como autoridad de identidad. Mantener emision JWT local contradice arquitectura y duplica riesgo de seguridad.

## Comandos Exactos

```bash
rg "generateToken|JwtService|jwt.secret" src/main/java src/main/resources -g "*"
./mvnw -DskipTests compile
./mvnw test -Dspring.profiles.active=test
```

## Ejemplo De Codigo (Si Aplica)

```txt
Claims claims = jwtParser.parseSignedClaims(token).getPayload();
String subject = claims.getSubject();
String role = String.valueOf(claims.get("role"));
```

## Verificacion

1. Auth ya no emite tokens locales para login.
2. Token firmado por Supabase se valida y extrae claims requeridos.
3. Compilacion y tests basicos de seguridad pasan.

## Resultado Esperado

Backend autenticado contra identidad externa unica, sin duplicar emision de credenciales.
