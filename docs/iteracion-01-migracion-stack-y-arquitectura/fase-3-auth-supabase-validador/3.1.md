# 3.1 - Crear estructura por capas para auth

## Objetivo

Reorganizar auth en estructura por capas para separar presentacion, aplicacion, dominio, persistencia e infraestructura.

## Razon Tecnica

La version actual de auth concentra responsabilidades. Esta tarea alinea el modulo con la adaptacion Java y prepara la migracion a Supabase JWT.

## Comandos Exactos

```bash
mkdir -p src/main/java/com/poo/miapi/module/auth/{presentation/{controller,dto},application/{usecase,service,dto},domain/{model,valueobject,event,policy},persistence/{repository,query,entity},infrastructure/{config,mapper,client}}
find src/main/java/com/poo/miapi/module/auth -maxdepth 3 -type d | sort
./mvnw -DskipTests compile
```

## Ejemplo De Codigo (Si Aplica)

```txt
module/auth/
  presentation/
  application/
  domain/
  persistence/
  infrastructure/
```

## Verificacion

1. Auth contiene las 5 capas con paquetes base.
2. Controladores nuevos viven en presentation/controller.
3. Compilacion sigue pasando tras reorganizacion.

## Resultado Esperado

Auth queda estructurado para desacoplar seguridad, validacion de token y reglas de aplicacion.
