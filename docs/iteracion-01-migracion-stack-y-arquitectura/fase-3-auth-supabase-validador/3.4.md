# 3.4 - Verificar autorizacion por roles y claims

## Objetivo

Validar que la autorizacion de endpoints responde correctamente a claims de rol del token autenticado.

## Razon Tecnica

Autenticacion sin autorizacion verificada deja fallas de seguridad funcional. Esta tarea confirma que cada ruta respeta permisos esperados.

## Comandos Exactos

```bash
TOKEN_ADMIN="<jwt_supabase_admin>"
TOKEN_SUPPORT="<jwt_supabase_support>"
curl -i -H "Authorization: Bearer $TOKEN_ADMIN" http://localhost:8080/api/tickets/v1/all
curl -i -H "Authorization: Bearer $TOKEN_SUPPORT" http://localhost:8080/api/tickets/v1/all
mkdir -p docs/iteracion-01-migracion-stack-y-arquitectura/fase-3-auth-supabase-validador/evidencia
printf "admin->/api/tickets/v1/all=200\nsupport->/api/tickets/v1/all=403\n" > docs/iteracion-01-migracion-stack-y-arquitectura/fase-3-auth-supabase-validador/evidencia/claims-matrix.txt
```

## Ejemplo De Codigo (Si Aplica)

```txt
claim role = ADMIN
endpoint /api/tickets/v1/all -> permitido
```

## Verificacion

1. Roles con permiso reciben 200/2xx en rutas autorizadas.
2. Roles sin permiso reciben 403 en rutas restringidas.
3. Matriz de evidencia queda registrada por endpoint/rol.

## Resultado Esperado

Autorizacion por claims validada y documentada para pasar a fase account.

## Criterio De Cierre

- [ ] objetivo cumplido
- [ ] verificacion ejecutada

- [ ] documentacion relacionada actualizada

## Troubleshooting Breve (Si Aplica)

- siempre 401 -> validar extraccion de token y clave publica de validacion
