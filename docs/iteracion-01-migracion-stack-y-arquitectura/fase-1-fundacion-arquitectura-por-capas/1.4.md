# 1.4 - Verificar reglas de capas y transacciones

## Objetivo

Validar que las reglas arquitectonicas base ya se respetan en la estructura nueva de la iteracion.

## Razon Tecnica

La arquitectura por capas solo aporta valor si se verifica. Esta tarea controla dos reglas criticas: transacciones en application y dominio sin dependencia Spring.

## Comandos Exactos

```bash
rg "@Transactional" src/main/java/com/poo/miapi/module -g "*.java"
rg "org.springframework" src/main/java/com/poo/miapi/module/*/domain -g "*.java"
rg "import com\.poo\.miapi\.module\..*\.repository\." src/main/java/com/poo/miapi/module/*/presentation -g "*.java"
./mvnw -DskipTests compile
```

## Ejemplo De Codigo (Si Aplica)

```txt
// permitido
@Service
public class TicketApplicationService {
    @Transactional
    public void execute(...) { ... }
}
```

## Verificacion

1. @Transactional aparece en application service, no en controller.
2. No hay imports de Spring dentro de paquetes domain nuevos.
3. Presentation no importa repositories directamente.

## Resultado Esperado

Las reglas minimas de arquitectura quedan comprobadas antes de migrar funcionalidad pesada.
