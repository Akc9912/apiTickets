# 1.1 - Definir estructura de paquetes por capas

## Objetivo

Crear la estructura base de paquetes por capas para los modulos migrados en Iteracion 01.

## Razon Tecnica

La adaptacion Java de engineering exige separacion explicita entre presentation, application, domain, persistence e infrastructure. Esta estructura evita mezclar responsabilidades.

## Comandos Exactos

```bash
mkdir -p src/main/java/com/poo/miapi/module/auth/{presentation/{controller,dto},application/{usecase,service,dto},domain/{model,valueobject,event,policy},persistence/{repository,query,entity},infrastructure/{config,mapper,client}}
mkdir -p src/main/java/com/poo/miapi/module/account/{presentation/{controller,dto},application/{usecase,service,dto},domain/{model,valueobject,event,policy},persistence/{repository,query,entity},infrastructure/{config,mapper,client}}
mkdir -p src/main/java/com/poo/miapi/module/ticket/{presentation/{controller,dto},application/{usecase,service,dto},domain/{model,valueobject,event,policy},persistence/{repository,query,entity},infrastructure/{config,mapper,client}}
find src/main/java/com/poo/miapi/module -maxdepth 3 -type d | sort
```

## Ejemplo De Codigo (Si Aplica)

```txt
module/ticket/
  presentation/
  application/
  domain/
  persistence/
  infrastructure/
```

## Verificacion

1. Cada modulo migrado contiene las 5 capas definidas.
2. No se crean carpetas fuera de la convencion establecida.
3. La estructura queda visible y verificable por comando find.

## Resultado Esperado

Existe base estructural uniforme para implementar codigo nuevo sin contaminar paquetes legacy.
