# 1.2 - Crear contratos internos entre modulos

## Objetivo

Definir contratos de aplicacion para comunicacion entre modulos sin dependencias directas a implementaciones internas.

## Razon Tecnica

Engineering define contratos claros entre capas y modulos. Esta tarea elimina el acoplamiento por imports directos y habilita evolucion independiente.

## Comandos Exactos

```bash
mkdir -p src/main/java/com/poo/miapi/shared/contracts/{auth,account,ticket}
touch src/main/java/com/poo/miapi/shared/contracts/auth/AuthContextContract.java
touch src/main/java/com/poo/miapi/shared/contracts/account/AccountQueryContract.java
touch src/main/java/com/poo/miapi/shared/contracts/ticket/TicketOwnershipContract.java
rg "package com\.poo\.miapi\.shared\.contracts" src/main/java/com/poo/miapi/shared/contracts -g "*.java"
```

## Ejemplo De Codigo (Si Aplica)

```txt
public interface AccountQueryContract {
    AccountProfileDto getProfileById(UUID accountId);
}
```

## Verificacion

1. Los contratos existen en shared/contracts por dominio.
2. Los contratos exponen interfaces, no implementaciones ni repositorios.
3. Los modulos consumidores pueden depender de contratos sin importar clases internas.

## Resultado Esperado

Comunicacion intermodulo desacoplada mediante interfaces versionables.

## Criterio De Cierre

- [ ] objetivo cumplido

## Troubleshooting Breve (Si Aplica)

- contratos con metodos ambiguos -> definir input/output explicito con DTOs o value objects
- duplicacion de contratos similares -> consolidar en un contrato por capacidad de negocio
