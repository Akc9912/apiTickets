# 2.3 - Aplicar cambios base de esquema y entidades

## Objetivo

Aplicar primera ola de cambios en esquema y entidades para soportar PostgreSQL + UUID en modulos priorizados.

## Razon Tecnica

La estrategia definida en 2.2 debe materializarse en DDL y entidades. Sin cambios reales de esquema la migracion no progresa.

## Comandos Exactos

```bash
mkdir -p docs/iteracion-01-migracion-stack-y-arquitectura/fase-2-migracion-db-y-uuid-base/evidencia
cp create_database.sql docs/iteracion-01-migracion-stack-y-arquitectura/fase-2-migracion-db-y-uuid-base/evidencia/create_database.before.sql
./mvnw -DskipTests compile
./mvnw test -Dspring.profiles.active=test
```

## Ejemplo De Codigo (Si Aplica)

```txt
id UUID PRIMARY KEY DEFAULT gen_random_uuid()
```

## Verificacion

1. Las entidades migradas usan UUID como identificador.
2. El esquema actualizado refleja tipos UUID en tablas objetivo.
3. Compilacion y tests basicos continúan en verde.

## Resultado Esperado

Base de datos y modelo de dominio avanzan en la direccion del stack objetivo sin corte global.
