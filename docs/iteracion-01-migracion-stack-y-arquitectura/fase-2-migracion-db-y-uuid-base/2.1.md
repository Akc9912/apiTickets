# 2.1 - Migrar dependencia y configuracion a PostgreSQL

## Objetivo

Alinear dependencias y configuracion de datasource al stack oficial (PostgreSQL) para eliminar dependencia activa de MySQL.

## Razon Tecnica

El stack base de engineering define PostgreSQL como persistencia principal. Mantener MySQL bloquea consistencia de entorno y migracion a UUID.

## Comandos Exactos

```bash
rg "mysql|postgresql" pom.xml src/main/resources/application.properties -g "*"
./mvnw -DskipTests compile
docker compose config
./mvnw -DskipTests compile
```

## Ejemplo De Codigo (Si Aplica)

```txt
spring.datasource.url=jdbc:postgresql://localhost:5432/apitickets
spring.datasource.driver-class-name=org.postgresql.Driver
spring.jpa.properties.hibernate.dialect=org.hibernate.dialect.PostgreSQLDialect
```

## Verificacion

1. pom.xml incluye driver PostgreSQL en runtime.
2. application.properties apunta a datasource PostgreSQL.
3. Compilacion completa sin errores por cambio de dependencia.

## Resultado Esperado

La aplicacion queda preparada para ejecutar sobre PostgreSQL en dev/staging.
