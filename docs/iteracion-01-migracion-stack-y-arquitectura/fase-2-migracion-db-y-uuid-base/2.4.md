# 2.4 - Verificar persistencia y repositorios post-migracion

## Objetivo

Verificar que los repositorios y consultas de los modulos migrados siguen funcionando luego de cambios a PostgreSQL y UUID.

## Razon Tecnica

Las roturas de persistencia suelen aparecer al final del cambio de tipo de ID. Esta tarea detecta incompatibilidades antes de avanzar a auth/account/ticket.

## Comandos Exactos

```bash
./mvnw -DskipTests compile
./mvnw test -Dspring.profiles.active=test
rg "JpaRepository<.*,\s*Integer>|\bint\s+getId\(" src/main/java/com/poo/miapi/module -g "*.java"
rg "MySQL|mysql" pom.xml src/main/resources src/main/java -g "*"
```

## Ejemplo De Codigo (Si Aplica)

```txt
public interface TicketRepository extends JpaRepository<TicketEntity, UUID> { }
```

## Verificacion

1. No quedan repositorios Integer en modulos migrados.
2. No quedan referencias activas a MySQL en configuracion objetivo.
3. Build y tests de persistencia pasan en entorno de prueba.

## Resultado Esperado

Persistencia consistente y lista para continuar con auth/account sobre base estable.
