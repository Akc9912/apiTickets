# 2.1 - Implementar bandeja support de mensajes

## Objetivo

Implementar bandeja operativa para que soporte consulte y filtre mensajes externos pendientes de decision.

## Razon Tecnica

Sin una bandeja de trabajo, el flujo externo queda ciego y obliga a operar directo sobre base de datos.

## Comandos Exactos

```bash
./mvnw -DskipTests compile
curl -s http://localhost:8080/v3/api-docs | rg "support|product"
./mvnw test -Dspring.profiles.active=test -Dtest='*Product*Support*Test'
```

## Ejemplo De Codigo (Si Aplica)

```txt
GET /api/v1/support/product/messages?status=IN_REVIEW&source=erp
GET /api/v1/support/product/messages/{id}
```

## Verificacion

1. Soporte puede listar mensajes por estado y origen.
2. La consulta incluye campos minimos para decision operativa.
3. Endpoints aplican autorizacion por rol support/admin.

## Resultado Esperado

Bandeja support disponible para gestionar mensajes externos sin procesos manuales.
