# 2.3 - Registrar historial operativo de acciones support

## Objetivo

Registrar historial de acciones de soporte sobre cada mensaje externo para mantener trazabilidad completa de decisiones.

## Razon Tecnica

Sin auditoria operativa, no se puede explicar por que un mensaje se resolvio, descarto o convirtio a ticket.

## Comandos Exactos

```bash
./mvnw -DskipTests compile
./mvnw test -Dspring.profiles.active=test -Dtest='*Audit*Test,*History*Test'
rg "audit|history|support" src/main/java/com/poo/miapi/module/product -g "*.java"
```

## Ejemplo De Codigo (Si Aplica)

```txt
Acciones auditadas:
- MARK_AS_RESOLVED
- DISCARD_MESSAGE
- CONVERT_TO_TICKET

Campos minimos:
- actorId
- actionType
- reason
- occurredAt
```

## Verificacion

1. Toda accion de soporte crea entrada de historial.
2. Historial se consulta por mensaje y por rango de fecha.
3. No se pierde trazabilidad en reintentos o errores.

## Resultado Esperado

Trazabilidad operativa completa para soporte y auditoria interna.

## Criterio De Cierre

- [ ] objetivo cumplido
- [ ] verificacion ejecutada
- [ ] evidencia registrada
- [ ] documentacion relacionada actualizada

## Troubleshooting Breve (Si Aplica)

- acciones sin historial -> revisar transaccion y orden de persistencia
- historial duplicado -> validar idempotencia de comando por requestId
