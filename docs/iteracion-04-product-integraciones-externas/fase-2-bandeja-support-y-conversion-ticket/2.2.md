# 2.2 - Implementar conversion controlada a ticket

## Objetivo

Permitir conversion de mensajes externos a tickets con reglas de negocio, validacion de duplicados y trazabilidad.

## Razon Tecnica

La conversion directa sin control puede generar tickets duplicados y ruido operativo.

## Comandos Exactos

```bash
./mvnw -DskipTests compile
./mvnw test -Dspring.profiles.active=test -Dtest='*Product*Conversion*Test,*Ticket*Application*Test'
rg "createTicket|TicketApplicationService|product" src/main/java -g "*.java"
```

## Ejemplo De Codigo (Si Aplica)

```txt
POST /api/v1/support/product/messages/{id}/convert-to-ticket
Body:
- category
- summary
- priority
- assignedTo (opcional)
```

## Verificacion

1. Cada mensaje puede convertirse una sola vez a ticket.
2. Ticket creado referencia el mensaje origen para auditoria.
3. Falla de validacion retorna error funcional explicito.

## Resultado Esperado

Flujo message -> ticket estable, controlado y sin duplicidad funcional.
