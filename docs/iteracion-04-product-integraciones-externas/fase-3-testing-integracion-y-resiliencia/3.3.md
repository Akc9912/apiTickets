# 3.3 - Ejecutar pruebas de integracion product-ticket

## Objetivo

Ejecutar pruebas de integracion extremo a extremo del flujo product -> soporte -> ticket para cerrar la iteracion.

## Razon Tecnica

Antes de avanzar a eventos/notificaciones, el flujo de integracion externa debe quedar estable y verificable.

## Comandos Exactos

```bash
./mvnw test -Dspring.profiles.active=test
mkdir -p docs/iteracion-04-product-integraciones-externas/fase-3-testing-integracion-y-resiliencia/evidencia
git status --short > docs/iteracion-04-product-integraciones-externas/fase-3-testing-integracion-y-resiliencia/evidencia/git-status-cierre.txt
```

Flujo objetivo:

## Verificacion

1. Suite de integracion principal queda en verde.
2. Evidencia tecnica de ejecucion queda registrada en fase.
3. No hay regresiones sobre endpoints existentes del MVP.

## Resultado Esperado

Iteracion 04 cerrada con flujo product-ticket estable y con evidencia reproducible.

## Criterio De Cierre

- [ ] objetivo cumplido
- [ ] verificacion ejecutada
