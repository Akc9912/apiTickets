# 3.2 - Implementar tests de idempotencia y reintentos

## Objetivo

Validar que reintentos de integraciones externas no generen duplicados ni estados inconsistentes.

## Razon Tecnica

Integraciones reales reintentan por timeout o errores transitorios; sin idempotencia se duplica la carga operativa.

## Comandos Exactos

```bash
./mvnw test -Dspring.profiles.active=test -Dtest='*Idempotency*Test,*Retry*Test,*Product*Integration*Test'
rg "idempot|retry|requestId" src/main/java/com/poo/miapi/module/product -g "*.java"
```

## Ejemplo De Codigo (Si Aplica)

```txt
Escenarios:
- mismo externalMessageId + mismo source -> no duplica registro
- retry tras timeout -> mantiene mismo estado final
- retry concurrente -> una sola conversion efectiva
```

## Verificacion

1. Duplicados quedan bloqueados por reglas de idempotencia.
2. Reintentos transitorios no crean tickets extra.
3. Estados finales son deterministas tras multiples intentos.

## Resultado Esperado

Flujo externo robusto ante reintentos y condiciones de red no ideales.
