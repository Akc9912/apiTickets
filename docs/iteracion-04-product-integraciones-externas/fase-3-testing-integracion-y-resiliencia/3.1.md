# 3.1 - Implementar tests de contrato API externa

## Objetivo

Construir pruebas de contrato para validar request/response de la API externa de product contra escenarios reales.

## Razon Tecnica

Los cambios de contrato son una de las principales causas de ruptura con integradores externos.

## Comandos Exactos

```bash
./mvnw test -Dspring.profiles.active=test -Dtest='*Product*Contract*Test'
curl -s http://localhost:8080/v3/api-docs > docs/iteracion-04-product-integraciones-externas/fase-3-testing-integracion-y-resiliencia/openapi-product.json
```

## Ejemplo De Codigo (Si Aplica)

```txt
Casos minimos de contrato:
- payload valido -> 202/201
- payload con campo obligatorio faltante -> 400
- api key invalida -> 401/403
```

## Verificacion

1. Casos de contrato principales quedan automatizados.
2. Respuestas de error son consistentes y documentadas.
3. OpenAPI generado refleja endpoints y esquemas actuales.

## Resultado Esperado

Contrato externo protegido ante regresiones y cambios involuntarios.

## Criterio De Cierre

- [ ] objetivo cumplido
- [ ] verificacion ejecutada
- [ ] evidencia registrada
- [ ] documentacion relacionada actualizada

## Troubleshooting Breve (Si Aplica)

- tests inestables por datos compartidos -> aislar fixtures por test
- diferencias entre OpenAPI y runtime -> regenerar docs con profile correcto
