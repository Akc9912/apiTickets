# 1.3 - Implementar autenticacion API key y validacion

## Objetivo

Proteger la API externa de product con API key y reglas de validacion de origen para reducir abuso e ingresos no autorizados.

## Razon Tecnica

Una API externa sin control de acceso se vuelve vector de spam y de entradas maliciosas que afectan operacion.

## Comandos Exactos

```bash
rg "SecurityFilterChain|OncePerRequestFilter|api key" src/main/java -g "*.java"
./mvnw -DskipTests compile
./mvnw test -Dspring.profiles.active=test -Dtest='*Security*Test,*Auth*Test'
```

## Ejemplo De Codigo (Si Aplica)

```txt
Headers requeridos:
- X-API-Key
- X-Source-System

Reglas minimas:
- key valida y activa
- source permitido
- requestId no vacio
```

## Verificacion

1. Requests sin API key reciben 401 o 403 segun politica.
2. Requests con key invalida no persisten mensajes.
3. Requests validas ingresan al flujo normal de procesamiento.

## Resultado Esperado

API externa autenticada y con validaciones basicas de seguridad operativa.
