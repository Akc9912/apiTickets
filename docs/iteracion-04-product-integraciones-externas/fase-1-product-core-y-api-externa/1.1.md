# 1.1 - Crear nucleo domain/persistence de product

## Objetivo

Crear la base del modulo product con capas domain/application/persistence y entidades de mensaje externo con estado de procesamiento.

## Razon Tecnica

Sin un nucleo de dominio y persistencia claro, la integracion externa termina acoplada al controller y se vuelve dificil de evolucionar.

## Comandos Exactos

```bash
mkdir -p src/main/java/com/poo/miapi/module/product/{domain,application,presentation,persistence,infrastructure}
rg "module/(auth|account|ticket)" src/main/java/com/poo/miapi/module -g "*.java"
./mvnw -DskipTests compile
```

## Ejemplo De Codigo (Si Aplica)

```txt
Entidad base sugerida:
- ProductExternalMessage
  - id (UUID)
  - externalSource
  - externalMessageId
  - payload
  - processingStatus (RECEIVED, IN_REVIEW, CONVERTED, DISCARDED)
  - receivedAt
```

## Verificacion

1. Existe estructura por capas en module/product.
2. El modelo de persistencia cubre mensaje externo y estado.
3. El proyecto compila sin romper modulos existentes.

## Resultado Esperado

Modulo product inicializado y listo para exponer API externa sin dependencia con codigo legacy.
