# 1.2 - Implementar API externa de ingreso de mensajes

## Objetivo

Implementar endpoint externo para recibir mensajes de sistemas terceros con contrato versionado y respuesta trazable.

## Razon Tecnica

El ingreso externo es la puerta principal del modulo product; necesita un contrato estable para evitar integraciones fragiles.

## Comandos Exactos

```bash
./mvnw -DskipTests compile
./mvnw test -Dspring.profiles.active=test -Dtest='*Product*Controller*Test,*Product*Api*Test'
curl -s http://localhost:8080/v3/api-docs | rg "product|external-message"
```

## Ejemplo De Codigo (Si Aplica)

```txt
POST /api/v1/product/external-messages
Headers:
- X-Source-System
- X-Request-Id
Body:
- externalMessageId
- payloadType
- payload
```

## Verificacion

1. Endpoint aparece en OpenAPI con request/response definidos.
2. Se valida payload minimo obligatorio antes de persistir.
3. Se retorna identificador interno para trazabilidad de soporte.

## Resultado Esperado

API externa funcional, documentada y utilizable por integraciones iniciales.
