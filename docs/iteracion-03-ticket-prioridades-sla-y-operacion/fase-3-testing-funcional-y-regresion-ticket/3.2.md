# 3.2 - Implementar tests funcionales de endpoints ticket

## Objetivo

Validar por tests funcionales los endpoints operativos de ticket por rol y estado.

## Razon Tecnica

El flujo ticket es la capacidad central del producto y debe mantener comportamiento consistente ante cambios.

## Comandos Exactos

```bash
./mvnw test -Dspring.profiles.active=test -Dtest='*TicketController*Test,*TicketApi*Test'
```

## Ejemplo De Codigo (Si Aplica)

```txt
escenarios:
- support crea ticket
- admin lista tickets
- rol no autorizado recibe 403
```

## Verificacion

1. Endpoints criticos de ticket tienen cobertura funcional minima.
2. Se validan codigos HTTP y contrato de respuesta.
3. Se validan permisos por rol en casos clave.

## Resultado Esperado

Cobertura funcional suficiente para detectar roturas de API en ticket.

- [ ] objetivo cumplido
