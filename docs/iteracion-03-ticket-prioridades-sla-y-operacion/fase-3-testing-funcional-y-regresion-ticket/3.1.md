# 3.1 - Implementar tests de reglas SLA

## Objetivo

Crear pruebas automatizadas para validar reglas de prioridad y umbrales SLA.

## Razon Tecnica

Las reglas de negocio son susceptibles a regresion cuando cambian configuraciones o estados de ticket.

## Comandos Exactos

```bash
./mvnw test -Dspring.profiles.active=test -Dtest='*Sla*Test,*Priority*Test'
```

## Ejemplo De Codigo (Si Aplica)

```txt
casos:
- riesgo SLA al 79% no escala
- riesgo SLA al 80% escala
- ticket resuelto no escala
```

## Verificacion

1. Tests cubren casos nominales y borde.
2. El resultado de cada regla es determinista.
3. Fallas de regla detienen pipeline.

## Resultado Esperado

Confianza en evolucion de reglas SLA sin romper operacion.

## Criterio De Cierre

- [ ] objetivo cumplido
- [ ] verificacion ejecutada
- [ ] evidencia registrada
- [ ] documentacion relacionada actualizada

## Troubleshooting Breve (Si Aplica)

- tests inestables por hora actual -> fijar clock de prueba
- exceso de mocks -> priorizar test de application con fixtures reales
