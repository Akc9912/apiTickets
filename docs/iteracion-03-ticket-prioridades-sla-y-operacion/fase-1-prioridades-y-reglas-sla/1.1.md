# 1.1 - Implementar modelo de prioridad en ticket

## Objetivo

Incorporar prioridad de ticket (CRITICAL, HIGH, MEDIUM, LOW) en dominio, persistencia y DTOs.

## Razon Tecnica

La operacion de soporte necesita clasificar tickets para responder segun impacto y urgencia.

## Comandos Exactos

```bash
rg "TicketStatus|Ticket" src/main/java/com/poo/miapi/module/ticket -g "*.java"
./mvnw -DskipTests compile
```

## Ejemplo De Codigo (Si Aplica)

```txt
enum TicketPriority { CRITICAL, HIGH, MEDIUM, LOW }
```

## Verificacion

1. El modelo ticket incluye campo priority.
2. DTO de creacion/respuesta acepta y expone prioridad.
3. Persistencia guarda prioridad sin romper datos existentes.

## Resultado Esperado

Clasificacion de tickets disponible para reglas SLA y operacion.
