# 1.2 - Definir configuracion SLA por prioridad

## Objetivo

Crear configuracion central de SLA por nivel de prioridad para controlar tiempos objetivo.

## Razon Tecnica

Sin configuracion explicita, las reglas de escalamiento quedan hardcodeadas y dificiles de operar.

## Comandos Exactos

```bash
./mvnw -DskipTests compile
```

## Ejemplo De Codigo (Si Aplica)

```txt
CRITICAL -> 2h
HIGH -> 8h
MEDIUM -> 24h
LOW -> 72h
```

## Verificacion

1. La configuracion SLA se almacena de forma administrable.
2. Cada prioridad tiene umbral definido y validado.
3. Se documenta politica de umbrales en README de fase.

## Resultado Esperado

Base de SLA configurable para evaluar y escalar tickets automaticamente.

## Criterio De Cierre

- [ ] objetivo cumplido
- [ ] verificacion ejecutada
- [ ] evidencia registrada
- [ ] documentacion relacionada actualizada

## Troubleshooting Breve (Si Aplica)

- conflicto entre config y reglas -> definir precedencia unica de lectura
- cambios operativos frecuentes -> permitir edicion sin redeploy
