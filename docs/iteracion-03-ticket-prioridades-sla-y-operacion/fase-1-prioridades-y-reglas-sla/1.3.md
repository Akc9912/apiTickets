# 1.3 - Aplicar reglas de evaluacion y cambio de prioridad

## Objetivo

Implementar reglas que ajusten prioridad o marquen riesgo de SLA segun estado y tiempos del ticket.

## Razon Tecnica

La prioridad inicial no siempre se mantiene; se necesita reevaluacion para sostener calidad de atencion.

## Comandos Exactos

```bash
./mvnw -DskipTests compile
./mvnw test -Dspring.profiles.active=test
```

## Ejemplo De Codigo (Si Aplica)

```txt
si consumo_sla >= 80% y estado != RESOLVED -> escalar prioridad
```

## Verificacion

1. Las reglas se ejecutan sobre datos de ticket y SLA.
2. Los cambios de prioridad quedan auditados.
3. No se generan bucles de escalamiento infinito.

## Resultado Esperado

Motor de reglas activo para prevenir incumplimientos de SLA.

## Criterio De Cierre

- [ ] objetivo cumplido
- [ ] verificacion ejecutada
- [ ] evidencia registrada
- [ ] documentacion relacionada actualizada

## Troubleshooting Breve (Si Aplica)

- escalamiento excesivo -> ajustar umbrales y ventana de evaluacion
- reglas no deterministas -> ordenar evaluacion por prioridad de regla
