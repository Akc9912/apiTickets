# 2.1 - Implementar asignacion operativa por prioridad

## Objetivo

Asignar tickets de forma operativa considerando prioridad, disponibilidad y rol.

## Razon Tecnica

La asignacion manual sin criterio de prioridad degrada tiempos de respuesta en escenarios criticos.

## Comandos Exactos

```bash
./mvnw -DskipTests compile
```

## Ejemplo De Codigo (Si Aplica)

```txt
CRITICAL/HIGH -> cola prioritaria
MEDIUM/LOW -> cola estandar
```

## Verificacion

1. Tickets criticos/altos entran en flujo de atencion prioritaria.
2. Se respeta disponibilidad/estado del asignado.
3. El historial de asignacion queda registrado.

## Resultado Esperado

Asignacion consistente con impacto del ticket y capacidad operativa.

## Criterio De Cierre

- [ ] objetivo cumplido
- [ ] verificacion ejecutada
- [ ] evidencia registrada
- [ ] documentacion relacionada actualizada

## Troubleshooting Breve (Si Aplica)

- ningun asignado disponible -> activar politica de cola y alerta a admin
- sobrecarga de un operador -> balanceo por capacidad max configurada
