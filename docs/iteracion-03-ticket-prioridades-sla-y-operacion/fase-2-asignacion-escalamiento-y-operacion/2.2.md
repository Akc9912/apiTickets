# 2.2 - Implementar escalamiento automatico por SLA

## Objetivo

Implementar escalamiento automatico cuando un ticket supera umbrales de riesgo SLA.

## Razon Tecnica

Sin automatizacion, los incumplimientos dependen de monitoreo manual y se detectan tarde.

## Comandos Exactos

```bash
./mvnw -DskipTests compile
./mvnw test -Dspring.profiles.active=test
```

## Ejemplo De Codigo (Si Aplica)

```txt
scheduler cada 30m -> evaluar tickets en riesgo -> escalar y notificar
```

## Verificacion

1. Scheduler detecta tickets en riesgo de SLA.
2. El ticket escala segun politica definida.
3. Se registra evidencia de accion automatica.

## Resultado Esperado

Incumplimientos reducidos por deteccion y accion temprana.

## Criterio De Cierre

- [ ] objetivo cumplido
- [ ] verificacion ejecutada
- [ ] evidencia registrada
- [ ] documentacion relacionada actualizada

## Troubleshooting Breve (Si Aplica)

- scheduler no corre en entorno -> revisar anotaciones y timezone
- escalamientos duplicados -> agregar control de idempotencia por ticket
