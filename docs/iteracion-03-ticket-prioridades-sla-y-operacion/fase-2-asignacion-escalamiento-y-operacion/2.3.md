# 2.3 - Exponer endpoints operativos y auditoria de cambios

## Objetivo

Exponer endpoints para gestion operativa de prioridad/SLA y registrar cada cambio con trazabilidad.

## Razon Tecnica

Operaciones de soporte necesitan visibilidad y control sin tocar base de datos manualmente.

## Comandos Exactos

```bash
./mvnw -DskipTests compile
curl -s http://localhost:8080/v3/api-docs | rg "priority|sla"
```

## Ejemplo De Codigo (Si Aplica)

```txt
PUT /api/v1/tickets/{id}/priority
GET /api/v1/tickets/sla-violations
```

## Verificacion

1. Endpoints operativos quedan documentados en OpenAPI.
2. Cada cambio relevante queda auditado.
3. Roles sin permiso reciben 403.

## Resultado Esperado

Control operativo seguro y trazable para prioridad y SLA.

## Criterio De Cierre

- [ ] objetivo cumplido
- [ ] verificacion ejecutada
- [ ] evidencia registrada
- [ ] documentacion relacionada actualizada

## Troubleshooting Breve (Si Aplica)

- endpoint sin control de permisos -> validar reglas de autorizacion por rol
- auditoria incompleta -> asegurar registro en cada comando de cambio
